## must_fix (3件)

### IA3-001: log-manager.ts のツール名マッピングが hardcoded

**場所**: `src/lib/log-manager.ts` line 94, 104, 140

log-manager.ts の3箇所でツール名を if-else チェーンで解決しているが、claude/codex/gemini の3種のみ対応。copilot 追加時に toolName が正しく解決されない。

```typescript
// line 94: claude/codex/gemini のみ対応
const toolName = cliToolId === 'claude' ? 'Claude Code' : cliToolId === 'codex' ? 'Codex CLI' : 'Gemini CLI';
```

**対応**: CLI_TOOL_DISPLAY_NAMES を import して使用するか、copilot ケースを明示追加。

### IA3-002: log-manager.ts listLogs() の 'all' ケースが3ツールのみ

**場所**: `src/lib/log-manager.ts` line 190

`listLogs()` で `cliToolId === 'all'` の場合に `['claude', 'codex', 'gemini']` のみ列挙。copilot を含む他ツールのログが一覧から漏れる。

**対応**: CLI_TOOL_IDS を import して使用する。

### IA3-003: response-poller.ts の 20箇所以上の分岐が網羅的に分析されていない

**場所**: `src/lib/polling/response-poller.ts` (20箇所以上)

特に completion detection (line 372-375) で copilot は既存のどの完了判定にも含まれず、レスポンス完了が検出されない:

```typescript
const isCodexOrGeminiComplete = (cliToolId === 'codex' || cliToolId === 'gemini' || cliToolId === 'vibe-local') && ...;
const isClaudeComplete = cliToolId === 'claude' && ...;
const isOpenCodeDone = cliToolId === 'opencode' && ...;
```

**対応**: 各分岐ポイントを個別にリストアップし、copilot の分類を Phase 1 調査後に決定。completion detection への copilot 追加は必須。
