CommandMate Sessions Flow

`Sessions` 系の画面遷移を整理したページです。特に PC では、`Sessions` は探索面、`Worktree Detail` は実行面、`Home/Review` は判断面として分担します。

前提 Sessions = 現行メイン画面の一覧部分
探索面 Sessions
実行面 Worktree Detail
判断面 Home / Review
Home
Waiting / Running summary → Sessions
Search / Filter / Select → Worktree Detail
Send / Watch / Files / Notes → Prompt / Review
Decision / Completion

1. 主要遷移

A. Home → Sessions

Home で “もっと探す” と判断したら Sessions へ。Waiting / Running の全件探索に使う。

B. Sessions → Worktree Detail

一覧から対象 worktree を選択。ここが最重要遷移。検索や絞り込みの終点。

C. Detail → Prompt response

Prompt は overlay / panel / sheet で処理可能。ただし文脈確認が必要なら detail 内で応答。

D. Detail → Review

完了した worktree は Review queue に上がり、差分確認や次の承認につながる。

2. 画面状態

Sessions / Empty

Repository 未登録なら Repositories へ誘導。ここで Add はしない。

Sessions / Filtered

Saved view と filters で対象を絞る。状態は URL or local storage で保持。

Detail / Running

History・Terminal・Files を使いながら監視。

Detail / Waiting

Quick reply 可能。重要判断は文脈付きで返す。

Detail / Done

Review へ送るか、その場で diff/log を開く。

3. Sessions 系遷移表

現在画面 ユーザー意図 遷移先 理由
Home 対象 worktree を探したい Sessions Home は検索用ではなく判断用のため
Home Waiting にすぐ答えたい Quick reply または Worktree Detail 軽い返答はその場、重い返答は文脈を見て detail
Sessions 対象を開いて実行したい Worktree Detail Send / Auto Yes / Files は detail の責務
Worktree Detail 完了結果を処理したい Review 未確認成果を一覧的に処理できる
Review 個別に深掘りしたい Worktree Detail 差分、ログ、会話履歴の確認には detail が必要
Sessions Repository を追加したい Repositories 登録・Sync は Sessions に混ぜない

4. 実装ルール

Rule 1

`Sessions` は対象選択まで。重い操作を持たせすぎない。

Rule 2

`Worktree Detail` は実行・監視・文脈確認の唯一の場所。

Rule 3

完了物や承認待ちは `Review` へ送る。Home はサマリーに留める。

Rule 4

Repository 追加や clone は `Repositories` に閉じ込める。
