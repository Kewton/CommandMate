## must_fix (2件)

### DR2-001: executeClaudeCommand() のコマンド名解決が cliToolId 直接渡し

**カテゴリ**: code_consistency
**影響セクション**: 3-2. コマンド形式の設計判断

設計方針書では `getCommandForTool()` マッピング関数の導入を提案しているが、現在の `claude-executor.ts` (line 157-158) は `execFile(cliToolId, args, ...)` と cliToolId をそのままコマンド名として渡している。copilot の場合、`cliToolId='copilot'` が `execFile('copilot', ...)` として実行され、`gh` コマンドとして実行されない。

**対応**: 実装チェックリストで `getCommandForTool()` を「新規ユーティリティ関数の追加」として明記し、`execFile(cliToolId, ...)` を `execFile(getCommandForTool(cliToolId), ...)` に変更するリファクタリングが必要であることを明示する。

### DR2-002: ALLOWED_CLI_TOOLS と CLI_TOOL_IDS の二重管理リスク

**カテゴリ**: code_consistency
**影響セクション**: 3-2. コマンド形式の設計判断

`claude-executor.ts` (line 37) の `ALLOWED_CLI_TOOLS` は `new Set(['claude', 'codex', 'gemini', 'vibe-local', 'opencode'])` とハードコードされており、`CLI_TOOL_IDS` からは導出されていない。`CLI_TOOL_IDS` の import すら存在しない。copilot 追加時に `ALLOWED_CLI_TOOLS` への追加を忘れるとスケジュール実行が拒否される。

**対応**: DR1-009 を must_fix に昇格させるか、同期を保証するテストケースを実装チェックリストに含める。

## should_fix (7件)

### DR2-003: getCliToolSkipPatterns() 関数は存在しない

設計方針書とIssue本文で `getCliToolSkipPatterns()` への copilot ケース追加が記載されているが、この独立関数は存在しない。skip patterns は `getCliToolPatterns()` の戻り値の `skipPatterns` プロパティとして提供される。

**対応**: 「`getCliToolPatterns()` の skipPatterns に copilot パターンを追加」に修正する。

### DR2-004: escapeShellArg() がコードベースに存在しない

セクション 3-4 のワンショットモデルコード例で参照されているが、コードベースのどこにも定義されていない。新規実装が必要。

### DR2-005: sendMessage() コード例が既存パターンと不一致

既存ツール（Codex, Gemini等）の sendMessage() は `detectAndResendIfPastedText()` を呼び出す共通パターンがあるが、設計書のコード例には含まれていない。

### DR2-006: interrupt() メソッドへの言及がない

BaseCLITool のデフォルト実装（Escape キー送信）を継承する想定であれば問題ないが、copilot のインタラクションモデルによっては動作しない可能性がある。Phase 1 前提調査に含めるべき。

### DR2-008: buildDetectPromptOptions() の copilot 対応方針が不明確

copilot のプロンプトインジケータに応じて `requireDefaultIndicator` の設定が必要になるが、設計書に指針がない。

### DR2-012: isInstalled() オーバーライドの import パターンが不明確
