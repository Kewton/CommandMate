## Must Fix（必須対応）

### MF-1: 再現手順が未記入

**カテゴリ**: 完全性
**場所**: ## 再現手順 セクション

**問題**:
テンプレートの番号リスト(1. 2. 3.)が空のまま残されています。再現手順がないとバグの検証ができません。

**推奨対応**:
以下の再現手順を追記してください:
1. モバイルブラウザでCommandMateにアクセス
2. 任意のWorktreeを選択
3. 下部タブバーで「files」タブをタップ
4. ファイル数が多いWorktreeでファイルツリー下部が切れて表示されないことを確認

### MF-2: 期待する動作と実際の動作が未記入

**カテゴリ**: 完全性
**場所**: ## 期待する動作 / ## 実際の動作 セクション

**問題**:

### MF-3: 概要にスクリーンショットのみで説明文がない

**カテゴリ**: 正確性
**場所**: ## 概要 セクション

**問題**:
テンプレートのプレースホルダー「何が起きたか簡潔に記述してください。」がそのまま残り、スクリーンショットだけが貼られています。問題の性質が文字情報として記録されていません。

**推奨対応**:
概要に以下を追記:
「モバイル版のfilesタブでファイル一覧が途中までしか表示されず、下方向にスクロールできない。原因はmainコンテナのpb-32(Tailwind)とpaddingBottom inline styleの二重適用(合計256px+)、およびoverflow-hiddenによるスクロール抑制。」

## Should Fix（推奨対応）

### SF-1: 根本原因の分析が記載されていない

**カテゴリ**: 技術的妥当性
**場所**: Issue本文全体

**問題**:
仮説検証で2つの根本原因が特定されていますが、Issue本文に記載がないため修正方針が定まりません。

**推奨対応**:
原因分析セクションを追加し、以下の2点を明記:

1. `WorktreeDetailRefactored.tsx` L1762でTailwind `pb-32`(128px)とinline style `paddingBottom: calc(8rem + env(safe-area-inset-bottom))`が重複適用されている
2. 同行の`overflow-hidden`がFileTreeView(`overflow-auto`)のスクロールを阻害している

**証拠**:
```tsx
// WorktreeDetailRefactored.tsx L1761-1765
<main
  className="flex-1 pb-32 overflow-hidden"
  style={{
    paddingBottom: 'calc(8rem + env(safe-area-inset-bottom, 0px))',
  }}
>
```
