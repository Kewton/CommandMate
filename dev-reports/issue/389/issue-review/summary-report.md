1. **saveContent()関数の設計問題解消**: useAutoSave用の保存関数（saveToApi）をisDirtyチェックを含まない純粋なAPI呼び出しとして切り出す方針が明確化された

3. **onSaveCompleteコールバックの矛盾解消**: auto-save成功時はsetOriginalContent()のみ実行し、onSave(filePath)はmanual save時のみ呼び出すことが明確化
4. **handleClose()の完全なauto-save対応**: isSaving中だけでなくisDirty=true（デバウンス待機中）でもsaveNow()を呼び出す
5. **テスト影響範囲の完全な列挙**: Save Operations(6件)、Unsaved Changes Warning(3件)、onClose callback(3件)、Ctrl+S関連テスト
6. **新UI要素のdata-testid命名定義**: auto-save-toggle、auto-save-indicator
7. **auto-save ON切り替え時のエッジケース対応方針**: initialValueRefによる未発火問題とsaveNow()での対処を明記
