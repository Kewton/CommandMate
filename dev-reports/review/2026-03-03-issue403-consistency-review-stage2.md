### should_fix (3件)

#### SF-1: mermaidダイアグラムのモード分岐欠落

- **カテゴリ**: セクション間の整合性
- **場所**: セクション2「アーキテクチャ設計」のmermaidダイアグラム
- **内容**: mermaidダイアグラムがデーモンモード経路（`nohup npm start >> server.log`）のみを表示しており、フォアグラウンドモード経路（`npm start`）が表現されていない。設計方針書はセクション1の補足およびセクション4-3で「フォアグラウンド/デーモン両モード共通」を強調しており、ダイアグラムの表現と矛盾する。
- **改善提案**: mermaidダイアグラムのビルド(H)ノード以降にデーモン/フォアグラウンドの分岐を追加する。例: `H --> J{daemon?} --> |Yes| I[nohup npm start >> server.log] / J --> |No| K[npm start (stdout)]`。「実行タイミング」テキスト説明欄では既に両モードが記載されているため、ダイアグラムをこれに揃える。

#### SF-2: Issue受入条件とのモード記述の乖離

- **カテゴリ**: Issue本文との整合性
- **場所**: 設計方針書全体 vs Issue #403 受入条件 #3
- **内容**: Issue本文の受入条件 #3 が「`scripts/build-and-start.sh --daemon` でのサーバー起動時にログローテーションが実行されること」とdaemonモードのみに言及しているが、設計方針書では「フォアグラウンド/デーモン両モード共通」と明記している。実装後にどちらが正しい受入条件なのか混乱する恐れがある。
- **改善提案**: 設計方針書に「Issue受入条件 #3 はdaemonモード起動時の検証を指すが、実装としてはデーモン分岐前に配置するため両モードで実行される（セクション1補足参照）」と注記を追加する。あるいはIssue側の受入条件を両モード対応に更新する。

#### SF-3: 関数コメント内の「|| true」表記と実装の不一致

- **カテゴリ**: 定数・変数名の整合性
- **場所**: セクション4-2「関数実装」コメント行
- **内容**: セクション4-2の関数コメントに `This function is called with "|| true" pattern` と記載されているが、セクション4-3の実際の呼び出しコードは `rotate_logs || echo "WARNING: ..." >&2` パターンを使用している。`|| true` は静かに成功扱いにする（出力なし）のに対し、`|| echo WARNING >&2` は警告を出力して成功扱いにする。動作は似ているがコメントの記述が不正確である。
- **改善提案**: コメントを `This function is called with a failure-safe pattern (see section 4-3).` に修正するか、`This function is called with "|| echo WARNING..." pattern` に正確に揃える。
