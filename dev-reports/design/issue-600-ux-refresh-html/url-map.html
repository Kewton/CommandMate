CommandMate URL Map

現行URLと、UX再設計で定義したURL一覧です。特に `/worktrees/:id` を中核の実行画面として残し、他画面との責務分離と遷移関係を整理しています。

維持するもの `/worktrees/:id` は残す

分離するもの 探索は `/sessions`、登録は `/repositories`

URL一覧

現行コードで存在するURLと、再設計で追加定義するURLを同じ表にまとめています。`Current` は現行実装、`Proposed` はUX設計上の定義です。

URL 状態 役割 `/worktrees/:id` との関係
/login Current 認証入口 認証後に `/` へ。間接的に detail へ入る前段。
/ Current → Proposed 現行は Repository + Worktree List + External Apps の混在。再設計後は Home / Mission Control。 detail へ入る前の判断面。待ち・完了・リスクから `/worktrees/:id` または `/worktrees/:id/reply` へ遷移。
/sessions Proposed worktree 一覧、検索、絞り込み、選択 対象を選んで `/worktrees/:id` に入るための探索面。
/repositories Proposed Local Path / Clone URL / Sync / Excluded restore 登録・同期後は `/sessions` または直接 `/worktrees/:id` に進む。
/review Proposed Done / Approval / Stalled の処理キュー 成果確認で detail を再度開く必要がある時に `/worktrees/:id` へ戻る。
/more Proposed Settings / External Apps / Help / Auth detail の補助設定面。通常は detail の直接前後には置かない。
/worktrees/:id Current + Core 実行・監視・返信・ファイル・ログ・ノート・Git の主画面 全導線の中心。再設計後も URL と責務を維持する。
/worktrees/:id/reply Proposed 軽量 Quick Reply 画面 / シート Home / Review / 通知から直接入る軽量導線。深掘り時は `/worktrees/:id` へエスカレーション。
/worktrees/:id?pane=history|terminal|files|logs|notes|git Proposed detail 内のタブ深リンク detail を壊さずに、現在できる操作をそのまま深リンク化するための定義。
/worktrees/:id/terminal Current フルスクリーン terminal detail の補助サブページ。主画面は `/worktrees/:id` のまま。
/worktrees/:id/files/:path Current フルスクリーン file viewer detail から派生する閲覧サブページ。閲覧後は detail に戻る。

結論

`/worktrees/:id` は残ります。しかも再設計ではむしろ重要度が上がります。`Home` や `Sessions` は「選ぶ」「判断する」画面で、`/worktrees/:id` は「実際に進める」画面です。

  • 今できる send / prompt reply / terminal / files / logs / notes / auto-yes は維持前提
  • 違いは、`/` に混在している情報を責務別URLへ分けること
  • Quick Reply は detail の代替ではなく軽量入口

`/worktrees/:id` を中心にした遷移関係

ポイントは、`/worktrees/:id` を「どこからでも戻ってこられる作業の母艦」にすることです。

/
Waiting / Running / Done の要約から detail へ
/sessions
一覧から対象 worktree を選び detail へ
/review
Done / Approval / Stalled から再確認のため detail へ
/worktrees/:id/reply
軽量返答で足りない時に detail へ移る
/repositories
登録・sync 後に sessions または detail へ
⇄
/worktrees/:id
実行・監視・返信・文脈確認の中心
/worktrees/:id/terminal
フルスクリーン terminal に派生
/worktrees/:id/files/:path
ファイル閲覧に派生

`/worktrees/:id` に入るべき時

  • Prompt の背景を見てから返答したい時
  • Terminal / Logs / Files / Notes を横断して判断したい時
  • 新しい指示を送る時
  • 完了結果の中身を確認して次アクションを決める時

`/worktrees/:id` に入らなくてよい時

  • Home で見える軽い yes/no の返答だけで足りる時 → `/worktrees/:id/reply`
  • 単に別の branch を探したい時 → `/sessions`
  • repository を登録したい時 → `/repositories`
  • 成果物を一覧で処理したい時 → `/review`

実装に落とす時のルール

  • `/worktrees/:id` は URL も責務も維持する
  • 現在の detail でできることは、detail 内タブや sub-route で維持する
  • 新URLは detail を置き換えるのではなく、前後の責務を整理するために増やす
  • Home から detail に飛ぶ時も、必ず `Repository 名 + Branch 名 + Agent + Status` を見せて文脈を失わない
